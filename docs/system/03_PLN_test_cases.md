# Сценарии тестирования (Test Cases)

## 1. Сценарий "Первый контакт"
**Описание**: Проверка логики добавления нового собеседника.

- **Действие**: Пользователь нажимает "Add Contact" и вводит PeerID друга.
- **Ожидаемый результат**:
    1.  В таблице `contacts` создается запись.
    2.  В списке чатов (Master Area) появляется новая строка.
    3.  При клике на строку открывается пустой Detail Area с заголовком PeerID.
    4.  Отправка первого сообщения инициирует создание потока libp2p.

## 2. Сценарий "Входящее сообщение"
**Описание**: Проверка реактивности UI и фоновой обработки.

- **Ситуация**: Приложение открыто на списке чатов, но выбран чат с Бобом. Приходит сообщение от Алисы.
- **Ожидаемый результат**:
    1.  `MessageProcessor` сохраняет сообщение Алисы.
    2.  Шина транслирует `EventNewMessage`.
    3.  `ChatListViewModel` видит событие, перемещает Алису в топ списка и обновляет счетчик непрочитанных.
    4.  `DirectChatViewModel` (Боба) игнорирует событие (так как `ChatID` не совпадает).
    5.  Если приложение свернуто, система (в будущем) выдает OS Notification.

## 3. Сценарий "Переключение между чатами"
**Описание**: Проверка производительности и отсутствия утечек данных.

- **Действие**: Быстрое переключение между 5-10 активными чатами в списке.
- **Ожидаемый результат**:
    1.  `DirectChatViewModel` корректно отменяет предыдущие подписки/контексты.
    2.  История сообщений загружается из SQLite за <100мс.
    3.  Память не растет бесконечно при каждом клике (корректная очистка binding-листов).
    4.  Индикаторы онлайна обновляются актуально для выбранного пира.
