# План реализации: Спринты (Aether)

## Спринт 1: Инфраструктура
**Цель**: Подготовить фундамент для хранения и передачи данных.

1.  **Event Bus**:
    - Реализация шины событий согласно `02_DES_event_bus_spec.md`.
    - Поддержка типизированных событий (`MessageEvent`, `PeerEvent`).
2.  **SQLite Schema**:
    - Обновление миграций (создание таблицы `contacts`).
    - Модификация таблицы `messages` (поля `chat_id`, `is_incoming`, `status`).
    - Создание индексов для быстрой выборки истории по `chat_id`.

## Спринт 2: Бизнес-логика
**Цель**: Обеспечить процессинг сообщений "под капотом".

1.  **Repository Layer**:
    - Обновление `MessageRepository` для работы с новыми полями.
    - Реализация `ContactRepository` для управления списком контактов.
2.  **MessageProcessor**:
    - Интеграция шины событий в процесс получения/отправки.
    - Логика "авто-создания" контакта при получении сообщения от нового PeerID.
    - Сохранение сообщений в БД с корректным `chat_id`.

## Спринт 3: Оживление UI
**Цель**: Полноценный реактивный интерфейс.

1.  **ViewModels**:
    - `ChatListViewModel`: Реактивное обновление списка при событии `NewMessage`.
    - `DirectChatViewModel`: Загрузка истории и "дописывание" новых сообщений налету.
2.  **UI Features**:
    - Реализация кнопки "Add Contact" (диалоговое окно для ввода PeerID).
    - Индикация статуса доставки (✓/✓✓) на основе событий из шины.
    - Автоматический скролл вниз при получении нового сообщения в активном чате.
