# Проектирование: UI Навигация (Aether)

## 1. Паттерн Master-Detail
Для Aether выбрана архитектура **Master-Detail**, так как это стандарт для десктопных мессенджеров.

- **Master Area (Слева)**: Список чатов (`ChatList`). Всегда доступен.
- **Detail Area (Справа)**: Окно переписки (`ChatView`). Обновляется при выборе чата.

### Почему не Stack?
Stack (замена экрана) менее удобен на десктопе, так как заставляет пользователя совершать лишние клики "Назад". Master-Detail позволяет быстро переключаться между контактами.

## 2. Управление состоянием (ViewModel)

### `MainViewModel` (Global State)
Содержит:
- `CurrentChatID`: `binding.String`. Хранит PeerID текущего выбранного чата.
- `AppStatus`: Состояние подключения ноды.

### Логика фильтрации событий
Когда шина событий транслирует `NewMessageEvent`:

1.  **Global Listener**: Обновляет список чатов (Master Area), поднимая чат наверх и обновляя счетчик непрочитанных.
2.  **Detail Listener**: 
    - Проверяет: `if event.ChatID == MainViewModel.CurrentChatID`.
    - Если `true`: Сообщение добавляется в `binding.List` текущего экрана чата, вызывая мгновенное появление "пузыря" и автоскролл.
    - Если `false`: Детальное окно игнорирует событие (оно отрисовано в фоне или закрыто).

## 3. Навигация в Fyne
1.  **Компоновка**: `container.NewHSplit(ChatList, ChatDetail)`.
2.  **Переключение чата**:
    - Пользователь кликает на элемент списка.
    - Вызывается `MainViewModel.SetCurrentChat(peerID)`.
    - `ChatDetail` подписывается на изменение `CurrentChatID` и очищает/загружает историю сообщений из SQLite.

## 4. Динамический скролл
При добавлении нового элемента в `ChatView`:
- Используется метод `list.ScrollToBottom()`.
- Вызов обрамляется в `fyne.DoMain(func() { ... })` для гарантии выполнения после обновления геометрии виджетов.
